# CS370

1. Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?

   For this project, I had to complete the building of an intelligent agent that uses deep Q-learning to solve a maze and find the treasure at the end of the maze.
   I was given starter code that included the maze environment in a .py file called TreasureMaze, the experience replay system called GameExperience.py, and parts of
   the neural network were already set up. My main responsibility was to finish the training logic in the qtrain() function, and I implemented the reinforcement
   learning process by adding in exploration and exploitation decision making, storing experiences, training the neural network with batches of data, and updating the
   target network. All of this combined allowed for the agent to learn the best path to reach the treasure from every starting position.

2. What do computer scientists do and why does it matter?

   Computer scientists are very important as they design and create software that we use daily to solve problems, and they do so by using algorithms, data, and programming.
   This work matters because it helps us automate tasks, improve the overall efficiency of said tasks, and solve complex problems in many important industries such as
   healthcare, business, and security. Without computer scientists, we would not have many of the modern technologies we rely so much on today.

3. How do I approach a problem as a computer scientist?

   Something important I've learned throughout my CS degree and this course is how I approach problems with the mindset of a computer scientist. I try to approach problems by
   first breaking them down into smaller steps and trying to understand how each part works. For example, in this project, I looked at the pseudocode and focused on one part
   of it at a time, like the action selection, training, and the performance tracking. I then test my code, look at the results, and make changes as needed if something isn't
   working the way I thought it would (or at all). Essentially, I follow a step-by-step process and this makes sure the solutions are correct and functional.

4. What are my ethical responsibilities to the end user and the organization?

   Another valuable lesson I've learned throughout my CS degree and this course is how I have an ethical responsibility to the users I am creating software for and the organization.
   My ethical responsibility is to create systems that are transparent, reliable, and most importantly, safe. I need to make sure that the system is working as intended and it does
   not or can not produce any results that can be harmful or misleading. It is also very important to communicate the limitations of systems like AI, since AI truly does not understand
   problems the same way as we do. For an organization, I have the responsibility to follow the best practices and protect their data (and the users), and build systems that can be
   trusted by both the organization and their users.
